Javascript SDK

Preparing the proof request

Create a request for proof generation in the browser

Quickstart

This is the simplest version of creating a proof request.

Import the library

import { ReclaimProofRequest } from '@reclaimprotocol/js-sdk';

Setup a request on the backend

const reclaimProofRequest = await ReclaimProofRequest.init(
    'APP_ID',
    'APP_SECRET',
    'PROVIDER_ID'
)
  • You can get the APP_ID and APP_SECRET using this guide.
  • The PROVIDER_ID is the proof you want the user to generate. You can see all the available providers here.

Start the session

await reclaimProofRequest.startSession({
    onSuccess: (proof) => {
        console.log('Verification successful', proof)
        // Handle the proof data
    },
    onError: (error) => {
        console.error('Verification failed', error)
    }
})

Get the request URL

const requestUrl = await reclaimProofRequest.getRequestUrl()
window.open(requestUrl, '_blank', 'width=500,height=700')

Using the in-browser SDK

For browser-only implementations without a backend, use the CDN version with these options:

Import from CDN

import { ReclaimProofRequest } from 'https://esm.sh/@reclaimprotocol/js-sdk@4.10.1';

Initialize with browser options

const reclaimProofRequest = await ReclaimProofRequest.init(
    'APP_ID',
    'APP_SECRET',
    'PROVIDER_ID',
    {
        useAppClip: false,
        customSharePageUrl: 'https://portal.reclaimprotocol.org'
    }
)
  • useAppClip: false — Forces web-based verification instead of iOS App Clip
  • customSharePageUrl — Routes to the web portal for in-browser verification

Common Errors

ReclaimProofRequest is not defined

Using old v3 syntax. The v4 SDK uses a static init() method:

// Wrong - v3 syntax
const request = new Reclaim.ProofRequest(APP_ID)

// Correct - v4 syntax
const request = await ReclaimProofRequest.init(APP_ID, APP_SECRET, PROVIDER_ID)

Verification opens App Store

Add useAppClip: false to the init options to force web flow.

Callbacks never fire

Call startSession() before getRequestUrl():

// Correct order
await reclaimProofRequest.startSession({ onSuccess, onError })
const url = await reclaimProofRequest.getRequestUrl()

Share URL redirects don't work

Set customSharePageUrl: 'https://portal.reclaimprotocol.org' in init options.

Popup blocked

Call window.open() directly inside the click event handler.

SDK Versions

  • v4.10.1 — Recommended, stable with portal support
  • v4.x — Use any latest 4.x version
  • v3.x — Deprecated, different API
